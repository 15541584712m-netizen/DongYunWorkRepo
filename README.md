##更新方式：日销看板##
1. 数据报表下载【日】流量数据，更新日支付金额、成功退款金额。注意成功退款金额需要把历史的也更新，不能只更新近一天的
2. 销售额拉取，复制三个品牌自营GMV和GSV

##更新方式：商品日看板##
1. 订单表下载对应时间的系统模板order、商品id模板
2. 数据处理：订单排退、排外部分销订单（保留内部小野兽 / 教练分销订单）、排除线下、内购、样品、代拍

##商品日看板初始口令##
想要看今年618和去年双11，同时期（比如说活动开始第一天、第二天... 以及累计天数）的产品GSV对比。收入的缺口，主要缺在了哪些产品上。也就是看看两次活动的销售额产品结构

4个表分别是两个活动的order表和goods表，可以用订单编号匹配。goods表里有商品id，可以作为商品维度的汇总。

order表需要进行筛选：
1、订单状态，去掉交易关闭
2、订单类型，去掉分销供货订单
3、商品名称去掉，含有任意：线下 内购 样品 代拍 FX
4、分销员所属分组，去掉【达人合作】
5、订单实付金额，去掉0元订单

先操作order表，然后再用goods表与之匹配，未匹配的订单就不看了，我们只看保留下来的订单。然后再看这些订单对应的商品的缺口。缺口要看每日维度的，以及截止到618最新一天的累计的

有一些商品618有，双十一没有，你也要标注出来。双十一有，618没有的也同理。
618还在进行中，但双十一已经结束了，你要把各个商品双十一期间总计金额也标注，并且增加一个618现在的完成进度条

25年双十一活动时间是25.10.10-31
26年618活动时间是26.5.8-31
